# 问卷、WTP 与 ABM 建模讨论总结

## 1. 核心问题

本轮讨论围绕两个基础问题展开：

1. 如果问卷已经能够测量居民的 WTP 或偏好，为什么还需要 ABM？
2. 由问卷塑造出的 Agent 是否真的能够可靠回答问卷本身回答不了的问题？

这两个问题直接关系到项目的研究定位、方法合理性和论文解释链。

## 2. 问卷与 ABM 的关系

问卷和 ABM 不应被理解为替代关系，而应被理解为不同层级的研究工具。

问卷主要回答：

```text
个体在特定情境下愿不愿意、多愿意、为什么愿意。
```

ABM 主要回答：

```text
许多具有异质性偏好的个体，在动态天气、电价波动、政策冲击和社会互动中反复决策时，系统可能如何演化。
```

因此，问卷提供的是微观参数和行为机制线索，ABM 用这些参数和机制进行动态系统实验。

## 3. 为什么不能只做问卷

如果论文目标只是估计：

```text
居民低碳产品 WTP 的均值、分布及影响因素
```

那么问卷加统计回归基本可以完成，ABM 的必要性较弱。

但如果论文目标是研究：

```text
极端天气、电价波动、政策干预和社会扩散如何共同塑造居民用电行为转型及 WTP
```

那么 ABM 有其方法价值，因为它能够处理问卷难以直接回答的问题：

- 微观偏好如何累积成宏观采用率或用电负荷变化；
- 补贴、惩罚和组合政策在群体层面是否互补或抵消；
- 社会网络是否加速低碳产品采用扩散；
- 极端天气反复出现后，家庭记忆和 WTP 是否产生路径依赖；
- 个体层面有效的政策，在系统层面是否产生不公平或反弹效应。

## 4. 关键限定：ABM 不是预测机器

需要明确的是：

```text
问卷制造的 Agent 不能自动可靠回答问卷回答不了的问题。
```

ABM 不是把问卷数据变成现实真相的机器。它更适合被定位为：

```text
基于实证参数的机制实验平台。
```

也就是说，ABM 回答的不是“现实未来一定会怎样”，而是：

```text
如果我们对个体行为机制的假设是这样的，那么在极端天气、政策冲击和社会网络作用下，系统可能如何演化。
```

## 5. ABM 结果可靠性的来源

ABM 的可信度不来自“Agent 是问卷生成的”这一点本身，而来自以下三方面：

1. **微观参数有实证来源**  
   问卷使 Agent 的收入、用电量、WTP、政策敏感度、社会影响易感性等参数不再完全凭空设定。

2. **行为规则有理论依据**  
   Agent 的决策机制应受到行为经济学和社会扩散理论约束，例如损失规避、框架效应、锚定效应、社会规范、有限理性和阈值采用。

3. **模型结论经过敏感性分析和校准**  
   不能只跑一个情景就宣称结论成立。应检查关键参数变化后，政策排序、扩散趋势和群体差异是否仍然稳健。

## 6. 当前项目的风险

如果项目采用以下路径：

```text
问卷 -> 填入 Agent 人设 -> LLM 自由推理 -> 输出宏观曲线
```

那么可靠性较弱，容易变成演示系统，而不是严肃的研究模型。

更稳妥的路径应是：

```text
问卷 -> 估计参数分布 -> 明确行为规则/LLM 约束 -> 多情景仿真 -> 敏感性分析 -> 与问卷或外部统计回验
```

## 7. 基于现有问卷，ABM 适合回答什么

### 较适合回答

- 不同收入、用电量、价格敏感度家庭的响应差异；
- 补贴、惩罚、极热持续时间对低碳产品采用意愿的方向性影响；
- 社会影响强弱对采用扩散速度的影响；
- 不同政策组合在模型假设下的相对效果排序。

### 需要谨慎回答

- 长期低碳产品采用率预测；
- 未来几年总电力负荷变化；
- 某政策实际节能量；
- 某城市真实市场渗透率。

这些问题需要外部统计数据或真实行为数据校准，否则只能作为情景模拟结果。

### 不宜直接宣称

- 某政策一定最优；
- 居民真实 WTP 就是某个精确数值；
- 社会扩散一定会达到某一比例；
- 模型能够准确预测现实能源转型。

## 8. 对论文表述的建议

较稳妥的论文定位是：

> 本研究基于问卷识别居民在极端天气和政策情境下的异质性偏好与 WTP 分布，并将其转化为 ABM 中的微观决策参数。ABM 不用于精确预测现实结果，而用于探索在不同政策组合和社会网络结构下，微观偏好如何可能累积为群体层面的用电行为转型路径。

这个定位能够避免把 ABM 夸大为预测工具，同时保留其在机制探索和政策情景比较中的价值。

## 9. 对项目推进的含义

后续项目推进应优先完成以下工作：

1. 明确 WTP 构念边界，区分低碳产品 WTP 与保供/可靠性 WTP。
2. 将问卷变量转化为可复现的 Agent 参数，而不是只生成模糊人设。
3. 约束 LLM Agent 的决策输出，使其服务于明确的行为机制。
4. 建立敏感性分析框架，检验结论是否依赖少数参数设定。
5. 尽可能引入外部数据，对用电量、价格、电器采用率或政策效果进行回验。

## 10. 一句话总结

问卷告诉我们“人可能怎么想”；ABM 探索“很多这样想的人在动态系统中反复互动后，系统可能怎么变”。但 ABM 的结论永远依赖假设，因此必须通过理论约束、参数校准、敏感性分析和外部回验来提高可信度。
