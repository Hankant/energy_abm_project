# 政策组合研究实验设计草案（V1.0）

## 0. 文档定位

这是一份面向“居民侧能源转型/需求响应/政策组合”研究的实验设计草案。  
设计目标不是再做一篇单纯的政策文本归类或宏观相关性研究，而是直接回应当前文献中的核心缺口：

- 现有研究已经较充分讨论“为什么要政策组合”，但缺少针对**居民侧真实政策包**的微观因果识别。
- 现有研究多关注单一政策工具，较少系统比较**价格、补贴、信息、自动化、公平保护**之间的交互作用。
- 中国语境下已有关于政策组合演进、嵌套试点与协同/权衡的研究，但缺少将这些宏观认识转化为**居民行为实验**和**可计算模拟**的工作。

本方案因此采用：

- 基于真实国家/地区政策包的原型化处理组设计
- 居民问卷随机实验
- 行为意向与公平评价并重的结果变量
- 进一步用实验参数驱动微观模拟/ABM

配套的政策包材料库见：[country_policy_mix_collection_materials.md](G:\Paperdownload\policy_mix_materials_20260408\country_policy_mix_collection_materials.md)

## 1. 为什么这个设计更符合当前研究缺口

与现有文献相比，这个设计的创新点主要在四个方面：

1. **研究对象下沉到居民侧。**  
   从 policy mix 理论、能源效率和双碳政策研究看，当前最薄弱的环节是“家庭如何面对政策组合而作出选择”。

2. **处理对象是政策包，而不是单一工具。**  
   这更符合 Rogge & Reichardt（2016）、Kern et al.（2019）和 Nieddu et al.（2024）所强调的“政策互动”视角。

3. **同时测量行为、接受度与公平。**  
   这回应了 Bessa et al.（2023）和居民侧 DSM 文献中对脆弱群体保护、能源贫困和政策正当性的强调。

4. **实验结果可直接进入模拟。**  
   它不是停留在主观态度，而是为后续政策包仿真提供微观参数输入。

## 2. 研究目标与研究问题

## 2.1 总体目标

识别不同居民侧政策组合及其实施顺序，如何影响：

- 政策接受度
- 需求响应参与意愿
- 允许自动化控制的意愿
- 感知公平与负担感
- 不同类型家庭之间的分布性结果

## 2.2 研究问题

1. 与单一价格政策相比，组合政策是否显著提高居民参与需求响应的意愿？
2. 哪类政策组合在居民侧更有效：
   - 价格 + 补贴
   - 价格 + 信息反馈
   - 价格 + 自动化
   - 价格 + 公平保护
   - 多工具一体化组合
3. 同样的政策包，在不同人群中是否存在显著异质性？
4. 政策组合的顺序是否影响政策效果？
5. 基于实验得到的行为参数，不同政策包在系统层面会产生怎样的峰荷、成本和公平差异？

## 4. 总体研究设计

本研究建议采用“两层设计”。

## 4.1 第一层：基于真实政策包原型的随机问卷实验

目标：识别居民对不同政策组合原型的接受度、行为意向和公平评价。

特点：

- 每个政策包都来自真实国家/地区政策方案，但在问卷中**不直接显示国家名称**，避免受访者因国家印象产生偏见。
- 所有政策包统一翻译成同一套属性语言，便于比较。
- 以 between-subject 随机分组为主，每个受访者只看一个主政策包。

## 4.2 第二层：微观模拟 / 轻量 ABM

目标：把第一层实验估计出的参与概率、自动化接受度、公平感和负担感参数，转化为系统层面的政策效果比较。

输出指标：

- 峰值负荷削减
- 总体参与率
- 低收入群体净收益
- 平均账单变化
- 政策成本
- 公平性指标

## 5. 样本设计

## 5.1 目标样本

建议首先以**中国城市居民家庭**为主要研究对象，因为：

- 你的研究问题最终更可能回到中国语境
- 中国已有较丰富的需求响应、分时电价、虚拟电厂和清洁取暖实践，可形成较强的政策落地讨论
- 中国研究目前最缺的恰恰是居民侧政策包的实验识别

## 5.2 建议样本量

建议初始样本：

- 总样本 `N = 2,400`
- 8 个处理组，每组约 `300`

如果预算允许，建议扩展为：

- 总样本 `N = 3,000`
- 对低收入群体、北方采暖家庭、租房群体、已有智能设备家庭适当过度抽样

## 5.3 建议分层

至少按以下维度配额或事后分层：

- 区域：东部 / 中部 / 西部 / 东北
- 气候：采暖主导 / 制冷主导 / 混合型
- 收入：低 / 中 / 高
- 住房产权：自有 / 租住
- 住宅类型：公寓 / 独栋 / 老旧住宅
- 家庭能源特征：是否电采暖、是否有 EV、是否有光伏、是否有储能

## 6. 处理组设计

## 6.1 设计原则

处理组不是简单比较单一工具，而是以真实政策包为基础构造“政策组合原型”。  
每个原型由五类属性构成：

- 价格机制
- 财政/账单支持
- 技术与自动化支持
- 保护与公平安排
- 实施方式与顺序

## 6.2 建议主处理组

| 组别 | 原型来源 | 价格机制 | 支持工具 | 自动化/信息 | 公平保护 | 顺序 |
|---|---|---|---|---|---|---|
| G0 | 控制组 | 固定单一电价 | 无 | 无 | 无 | 无 |
| G1 | 英国型 | 双时段或分时电价可选 | 冬季账单折扣 + 住宅升级支持 | 智能电表可接入 | 自动识别低收入家庭 | 先支持后升级 |
| G2 | 加州型 | 可选动态小时电价 | 需求响应奖励 + 低收入电费折扣 + 社区清洁电力 | 聚合商/需求响应参与 | CARE/FERA 类定向保护 | 同步实施 |
| G3 | 澳大利亚型 | TOU/灵活电价 | 家庭电池补贴/升级融资 | 智能家居自动控制 + 实时信息 | 零售商主动识别补贴资格 | 技术先行 |
| G4 | 欧盟型 | 固定价与动态价合同并存 | 可并行签订不同合同 | 清晰账单与合同信息 + energy sharing | 脆弱用户防断供保护 | 合同选择优先 |
| G5 | 中国清洁取暖型 | 居民峰谷分时电价 | 电采暖补贴 + 免费更换电表 + 能效改造配套 | 单独计量 | 民生保障优先 | 先设施后电价 |
| G6 | 中国虚拟电厂型 | 峰时错峰激励或响应补贴 | 社区或园区参与奖励 | 自动调节公共/可控负荷 + 平台聚合 | 保障居民基本舒适和重点用电 | 先试点后推广 |
| G7 | 优化组合型 | 可选 TOU/动态价 | 改造补贴 + 账单保护 + 响应奖励 | 智能电表 + 实时反馈 + 自动控制可选 | 低收入定向补偿 + 默认保护 | 明确“先支持后提价” |

## 6.3 处理组呈现方式

在问卷中，不直接告诉受访者“这是英国/加州方案”，而统一写成：

> 假设你所在城市将在未来一年推出一项新的居民用电政策方案。以下是方案主要内容……

这样可以减少国家印象、制度偏好和地缘态度的噪音。

## 6.4 可选扩展：顺序实验

如果样本量足够，可以将 G7 再拆成三个顺序组：

- G7a：先补贴/改造，后价格调整
- G7b：价格和支持同步推出
- G7c：先价格调整，后补贴/改造

这样可以直接识别“顺序效应”，这正是当前研究中很少被系统检验的部分。

## 7. 问卷结构与题项设计

## 7.1 问卷模块结构

建议问卷分成 6 个模块：

1. 基础人口与住房信息
2. 家庭能源使用特征
3. 数字能力与政策信任
4. 随机政策情景
5. 政策反应问题
6. 行为任务 / 选择题

## 7.2 基础信息模块

建议题项：

- 您的年龄、性别、教育程度、就业情况
- 家庭人口数
- 月收入/年收入区间
- 是否有老人、小孩、长期居家成员
- 住房产权：自有 / 租住 / 保障性住房
- 住宅类型：高层公寓 / 多层住宅 / 独栋 / 农村自建房
- 建成年代：新建 / 一般 / 老旧

## 7.3 家庭能源特征模块

建议题项：

- 家庭主要采暖方式：集中供暖 / 电采暖 / 燃气 / 其他
- 夏季制冷设备情况
- 是否拥有以下设备：
  - 智能电表
  - 可联网空调
  - 电热水器
  - 电动车/充电桩
  - 光伏
  - 家庭电池
- 电费账单感知：很轻松到很吃力
- 是否愿意通过调整用电时间节约电费

## 7.4 数字能力与政策信任模块

建议量表题项（1-7 分）：

- 我愿意使用手机 App 或在线平台管理家庭用电
- 我能理解电价随时段变化的规则
- 我信任电网/供电公司正确结算电费
- 我信任政府推出居民用能政策时会兼顾公平
- 我担心智能设备会侵犯隐私
- 我愿意授权设备在不影响基本舒适的情况下自动调节

## 7.5 情景后的核心结果变量

受访者阅读随机情景后，建议至少设置以下题项。

### 政策接受度

- 我支持这个政策方案
- 如果当地实施，我愿意参加
- 我愿意推荐家人/邻居参与

### 行为意向

- 我愿意把洗衣、热水、充电等高耗电行为转移到低价时段
- 我愿意为此安装或升级智能设备
- 如果有补贴，我愿意做家庭能效改造
- 如果能节省账单，我愿意接受一定程度的自动控制

### 公平与负担

- 这个方案对普通家庭是公平的
- 这个方案能较好保护低收入家庭
- 这个方案可能让我的家庭负担增加
- 我担心这个方案会让部分家庭“被迫参与”

### 信任与可实施性

- 我相信该方案能够被清楚、透明地执行
- 我相信账单结算会清楚明白
- 我认为这个方案操作起来不复杂

### 舒适与风险

- 我担心这个方案会影响居住舒适度
- 我担心高峰期自动调节会影响生活安排
- 我担心设备或平台出现技术故障

## 7.6 行为任务建议

为了避免只有态度题，建议在问卷后设置一个简短行为任务。

### 任务 A：账单-舒适度权衡题

给出一个简短场景：

- 若在晚高峰降低 1.5℃ 空调设定 1 小时，可节省 X 元
- 若把电动车改到低谷时段充电，可节省 X 元
- 若参与自动响应，每月返还 X 元

要求受访者做选择：

- 接受
- 不接受
- 只有在补贴更高时接受

### 任务 B：双方案选择题

呈现两个政策包：

- 方案 A：电价更灵活，但补贴更多
- 方案 B：电价更稳定，但支持较少

要求选择更偏好的方案。

这类 choice task 可以用于构造更接近行为的选择数据。

## 8. 可检验假设

### H1：组合优于单一

相较于单一价格政策，含有补贴、信息反馈、自动化或公平保护的政策组合，会显著提升政策接受度与参与意愿。

### H2：公平保护提升接受度

在相同价格机制下，加入低收入家庭账单保护或定向补贴，将显著提高整体样本和低收入样本的支持度。

### H3：自动化需要信任支持

自动化控制类政策包只有在受访者对实施主体具有较高信任，或附带明确透明说明时，才更容易被接受。

### H4：响应能力决定价格政策效果

住房能效较差、设备基础较弱或租房家庭，对单一价格政策响应较弱；但对“价格 + 改造支持”组合更敏感。

### H5：顺序重要

“先补贴/先改造，后价格调整”的政策顺序，比“先价格调整，后补贴”更能提升接受度、降低负担感。

### H6：社区聚合比纯价格机制更可能提高中国居民接受度

在中国语境下，若以社区、公共服务或虚拟电厂聚合的方式组织需求响应，并设置经济补偿和舒适边界，接受度将高于纯粹的价格惩罚型方案。

### H7：低收入群体对政策包差异更敏感

低收入家庭对账单保护、补贴和自动资格识别的敏感度高于中高收入家庭。

### H8：带有“支持 + 保护 + 灵活价格”的综合政策包，在公平和效率两个维度上更有可能同时占优

这是假设最终模拟中最值得检验的综合命题。

## 9. 估计策略

## 9.1 第一层实验分析

建议使用：

- OLS / Ordered Logit：分析支持度、公平感、负担感
- Logit / Probit：分析是否愿意参与、是否愿意接受自动控制
- Heterogeneous treatment effect models：分析不同收入、住房和设备条件下的处理效应

关键交互项：

- 政策包 × 收入
- 政策包 × 住房产权
- 政策包 × 是否电采暖
- 政策包 × 是否拥有智能设备
- 政策包 × 信任水平

中介机制可检验：

- 政策包 → 感知公平 → 支持度
- 政策包 → 账单预期 → 参与意愿
- 政策包 → 信任 → 自动化接受度

## 9.2 可选扩展：属性分解 conjoint

如果样本和时间允许，建议在主实验后附加一个小型 conjoint 模块，用于识别各属性的边际效应。  
可设置以下属性：

- 电价结构：固定 / TOU / 动态
- 支持方式：无 / 一次性补贴 / 持续账单返还
- 自动化：无 / 可选 / 默认可用
- 保护机制：无 / 低收入保护 / 普惠 + 低收入定向
- 参与方式：自愿申请 / 自动识别 / 默认加入可退出
- 实施顺序：同步 / 支持优先 / 价格优先

这一步有助于把原型比较进一步拆解成属性级证据。

## 10. 模拟思路

## 10.1 为什么需要模拟

实验可以告诉我们：

- 居民支持什么政策包
- 哪类人更愿意参与
- 哪些组合被认为更公平

但它不能直接告诉我们：

- 系统峰值到底能降多少
- 哪个政策包的财政成本更低
- 哪些群体最后是净受益者或净受损者

因此建议增加微观模拟或轻量 ABM。

## 10.2 模拟框架

### 输入参数

来自实验估计：

- `p_join(g, x)`：不同群体在政策包 `g` 下的参与概率
- `p_auto(g, x)`：接受自动控制的概率
- `shift_kwh(g, x)`：愿意转移的高峰负荷比例
- `retrofit_takeup(g, x)`：改造或设备升级意愿
- `fairness_penalty(g, x)`：对被认为不公平方案的参与折减

其中 `x` 代表家庭特征：

- 收入
- 住房
- 设备基础
- 气候
- 信任水平

## 11. 预期贡献

如果这套设计顺利执行，理论和方法上至少有四个贡献：

1. 将 policy mix 理论真正推进到居民侧微观行为层面。
2. 在中国语境下，把宏观政策组合研究与居民行为实验打通。
3. 识别哪些组合更有效，以及“对谁有效”。
4. 为政策制定提供可计算的“效率—公平—可接受性”联合评估框架。

## 12. 推荐执行顺序

建议按以下步骤推进：

1. 先完成国家/地区政策包材料库与编码。
2. 把处理组问卷情景写成统一格式。
3. 做 50–100 人预测试，检查题项理解。
4. 正式发放主实验问卷。
5. 估计处理效应与异质性。
6. 将估计参数接入微观模拟。
7. 输出政策建议：哪些政策包最平衡、哪些只适合特定群体、哪些顺序更优。

## 13. 一句话版本的研究设计摘要

> 本研究拟基于英国、加州、澳大利亚、欧盟及中国典型居民侧政策包，构建一套面向中国家庭的随机情景实验，识别不同政策组合及其实施顺序对居民需求响应参与意愿、感知公平和自动化接受度的影响，并进一步将实验参数输入微观模拟模型，以比较不同政策包在削峰、账单负担和分配公平上的综合效果。
